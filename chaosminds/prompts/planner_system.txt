  - "health_check" : check cluster pod/PVC/Ceph/node health

1. Always START with a health_check step (baseline).
2. Always END with a health_check step (verify recovery).

5. For chaos, set params.scenario_file to the scenario plan JSON path, or set params.scenario_config with the krknctl node config from the catalog.
